# Mirage-Sentinel Highlights

## Highlight 1: Active API Deception Flow

I built Mirage-Sentinel as a financial API defense prototype that separates normal traffic from suspicious requests before they reach the real banking demo. High-risk traffic is routed into the Mirage deception path, where fake sessions, accounts, transactions, cards, and bills can keep the attacker engaged without exposing real business data.

## Highlight 2: SOC Visibility and Forensic Logging

I added the logging and dashboard foundation needed for security monitoring. The system records traffic decisions, risk details, Mirage outcomes, session chain identifiers, and timing data so a SOC user can review recent activity, replay attack chains, and measure how effective the deception path is.
